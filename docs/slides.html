appteam is a Go CLI tool that generates complete multi-agent team configurations for Claude Code projects. One command, eight questions, and you have a fully structured team ready to go.

The Problem

  • No standard team structure across projects
  • Pipeline rules get lost or ignored
  • Skills recreated for every project
  • Agent roles are ad-hoc — unclear responsibilities
  • Backlog tracking is optional when it should be mandatory
Managing AI coding agents is chaotic without structure. Every project reinvents agent roles, workflow rules, and skill commands from scratch. This leads to inconsistency, missed steps, and lost context.

The Solution

$ appteam
  • Interactive wizard — complete team setup in seconds
  • Opinionated structure that enforces best practices
  • Consistent across projects — same roles, pipeline, conventions
33+ Generated Files
0 Dependencies
One command. Eight questions. A complete multi-agent team with roles, skills, workflows, and tracking. Pure Go, zero external dependencies. Works with any Claude Code project.

Everything a Claude Code project needs to run a structured multi-agent team. Agent definitions, skill commands, project documentation, and saved settings for future regeneration.

Team Sizing

Lean PM + 1 SWE + SWE-Test — quick fixes, small projects
Standard PM + TPM + 2 SWEs + SWE-Test + Reviewer — the default
Full All agents, up to 5 SWEs — large milestones

+ Custom agents: DBA, UX Designer, Tech Writer — define your own

Three team sizes to match your project's needs. Lean for solo work, Standard for most projects, Full for complex multi-stream work. Plus custom agent roles for anything else.

The Pipeline

PO → PM → TPM → SWE → Test → Review → Release
  • Every feature gets a spec before coding starts
  • Every piece of work tracked in the backlog
  • Every milestone has release notes and a git tag
  • No shortcuts — the pipeline is mandatory
The pipeline is non-negotiable. Every piece of work follows this flow: PO provides feedback, PM writes a spec, TPM assigns work, SWEs implement, testing verifies, review checks quality, then release. Backlog and progress are updated at every step.

How It Works

$ go install github.com/ahafin/appteam@latest

$ cd my-project
$ appteam

━━ Step 1 of 8 ━━ Project Basics
▸ Project name: my-app
▸ Description: A REST API for widgets
▸ Tech stack: Go, PostgreSQL

━━ Step 5 of 8 ━━ Agent Team
▸ Team size (lean/standard/full): standard
▸ Number of SWE agents (1-5): 2

✓ Generated 33 files in ./
✓ Settings saved to .appteam/settings.json

Regenerate anytime: appteam -r  |  Target dir: appteam -d ./project

Install with go install, run the wizard, answer 8 steps. Settings are saved for regeneration. Use -r to regenerate without the wizard, -d to target a different directory.
