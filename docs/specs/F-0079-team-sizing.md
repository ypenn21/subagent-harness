# F-0079: Team Sizing — Lean / Standard / Full Team Configurations

**Type:** Feature
**Priority:** P1 (important)
**Status:** Draft
**Requested by:** PO
**Date:** 2026-03-17

## Problem

appteam currently generates a fixed "standard" team structure (PM + TPM + 2 SWEs + SWE-Test + Reviewer) with a full mandatory pipeline. This one-size-fits-all approach is a poor fit for two common scenarios:

1. **Small tasks** — Quick fixes, solo features, and minor changes don't need a TPM, Reviewer, or multiple SWEs. The overhead of the full pipeline (10+ steps, 6 agents) slows down work that a single developer could handle with minimal coordination. A lean team (PM + 1 SWE + SWE-Test) with a simplified pipeline is more appropriate.

2. **Large milestones** — Multi-feature releases, architectural changes, and cross-cutting concerns benefit from additional agents (up to 5 SWEs, SWE-QA, Platform) that aren't generated by default. Teams doing large-scale work must manually enable optional agents and still don't get pipeline rules tuned for parallel execution at scale.

A configurable team size (lean, standard, full) lets the PO choose the right team structure for the scope of work, with the generated CLAUDE.md pipeline rules automatically adapting to match.

This feature also incorporates F-0080 (backlog enforcement), ensuring that `docs/BACKLOG.md` tracking is mandatory in all team sizes — lean teams don't get to skip the backlog just because they have fewer agents.

## Requirements

### Config

1. Add a `TeamSize string` field to `ProjectConfig` in `internal/config/config.go` with valid values: `"lean"`, `"standard"`, `"full"`
2. Update `backfillDefaults` logic in `internal/config/settings.go` to default `TeamSize` to `"standard"` when the field is empty or missing, ensuring backward compatibility with pre-v0.12.0 `settings.json` files

### Wizard

3. Add team size selection to wizard Step 5 (Agent Team) as the **first question** in the step, before SWE count or optional agent prompts:
   ```
   Team size (lean/standard/full) [standard]:
   ```
   Display a brief explanation of each option when the step header is shown:
   ```
   lean     — PM + 1 SWE + SWE-Test (quick fixes, small features)
   standard — PM + TPM + 2 SWEs + SWE-Test + Reviewer (default)
   full     — PM + TPM + up to 5 SWEs + SWE-Test + SWE-QA + Reviewer + Platform
   ```

4. **Lean** team size behavior in wizard:
   - Skip SWE count prompt — hardcode to 1 SWE
   - Skip optional agent prompts (Platform, Reviewer, SWE-QA) — none are generated
   - Skip SWE-Test prompt — SWE-Test is always included
   - Proceed directly to custom agents prompt (custom agents are still allowed in lean mode)
   - Proceed to model selection as normal

5. **Standard** team size behavior in wizard:
   - Current behavior unchanged — prompt for SWE count (default 2), optional agents (Platform, Reviewer, SWE-Test, SWE-QA), custom agents, model selection
   - SWE-Test defaults to Y (current behavior)

6. **Full** team size behavior in wizard:
   - SWE count defaults to 5 (instead of 2)
   - All optional agents (Platform, Reviewer, SWE-Test, SWE-QA) default to Y and are enabled automatically — skip prompts or show them as auto-enabled
   - Proceed to custom agents and model selection as normal

### CLAUDE.md Template

7. The CLAUDE.md template in `internal/templates/claude_md.go` must include conditional pipeline rules based on `TeamSize`:

   **Lean pipeline** (when `TeamSize == "lean"`):
   - No TPM role — PM handles backlog, progress, and release notes directly
   - Pipeline steps simplified:
     1. PO provides feedback to PM
     2. PM creates spec in `docs/specs/`
     3. PM creates/updates work items in `docs/BACKLOG.md`
     4. PM assigns work to SWE-1
     5. SWE-1 implements on feature branch
     6. SWE-1 hands off to SWE-Test for verification
     7. SWE-1 updates `docs/BACKLOG.md` marking items complete
     8. PM updates `docs/PROGRESS.md` and `docs/RELEASENOTES.md`
     9. PM reports back to PO
     10. Tag the release after PO approval
   - Roles section lists only: PO, PM, SWE-1, SWE-Test
   - PM role description expanded: "Receives PO feedback. Translates to specs. Manages backlog, progress log, and release notes. Coordinates with SWE-1 and SWE-Test."

   **Standard pipeline** (when `TeamSize == "standard"`):
   - Current pipeline rules unchanged (PM, TPM, SWE-1/2, SWE-Test, Reviewer)

   **Full pipeline** (when `TeamSize == "full"`):
   - Current pipeline rules with all optional agents described
   - SWE count up to 5 documented in roles
   - SWE-QA and Platform roles described in the roles section

8. **Backlog enforcement (F-0080):** Regardless of team size, the CLAUDE.md template must include an explicit, prominent rule:
   ```
   ### Backlog Tracking (Non-Negotiable)

   **Every piece of work gets a backlog entry in `docs/BACKLOG.md` — no exceptions.**
   Regardless of team size, all features, bug fixes, and enhancements must be tracked
   in the backlog before implementation begins and updated when completed.
   ```
   This section must appear in the CLAUDE.md output for lean, standard, and full team sizes.

### Generator

9. The generator in `internal/generator/generator.go` must conditionally generate agent files based on `TeamSize`:

   | Agent File | Lean | Standard | Full |
   |-----------|------|----------|------|
   | PM | Yes | Yes | Yes |
   | TPM | No | Yes | Yes |
   | SWE-1 | Yes | Yes | Yes |
   | SWE-2 through SWE-5 | No | Per config | Per config (default 5) |
   | SWE-Test | Yes | Per config | Yes |
   | Reviewer | No | Per config | Yes |
   | SWE-QA | No | Per config | Yes |
   | Platform | No | Per config | Yes |
   | Custom agents | Yes | Yes | Yes |

10. When `TeamSize == "lean"`, the generator must:
    - Generate exactly 1 SWE file (SWE-1)
    - Skip TPM, Reviewer, SWE-QA, Platform agent files
    - Always generate SWE-Test
    - Override any config values that conflict (e.g., if `SWECount` is > 1 but team is lean, generate only 1)

### Version

11. Version bump to `0.12.0` in `main.go`

### Tests

12. New tests must cover:
    - Config: `TeamSize` field serialization/deserialization, backfill default to "standard"
    - Wizard: team size prompt parsing, conditional agent prompt skipping for lean, default overrides for full
    - Templates: CLAUDE.md rendering with each team size (lean/standard/full), verifying correct pipeline rules and backlog enforcement section present in all
    - Generator: conditional agent file generation for each team size, lean generates only PM/SWE-1/SWE-Test, full generates all agents

## Out of Scope

- Dynamic team resizing during a session (changing team size requires re-running the wizard or regenerating with `-r`)
- Per-agent model selection based on team size (model selection remains uniform across all agents)
- Team size as a CLI flag (must go through wizard or `settings.json`)
- Hybrid team sizes or custom agent counts beyond the three presets (custom agents cover this need)
